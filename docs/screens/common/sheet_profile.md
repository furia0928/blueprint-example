# 프로필 설정

> 트리거: 헤더 프로필 아바타 클릭 (프로젝트 목록·화면 상세 등 모든 페이지 공통)
> 상세: [features/AUTH.md](../../features/AUTH.md), [DB.md](../../DB.md)

## 화면 구성

우측 슬라이드 시트. 내부 사용자와 게스트(뷰어)가 동일한 시트를 사용하되, 역할별로 편집 가능 범위가 다르다.

### 프로필 영역

**elements**
- 내부 사용자: GitHub 아바타 + 사용자명
- 게스트: 빈 아바타(이니셜 또는 빈 원) + 이메일 (TODO: 게스트 전용 아바타)
- 아바타와 사용자명은 GitHub에서 가져오므로 플랫폼에서 수정 불가

### 기본 정보 편집

**elements**
- 텍스트 항목: 이름 (display_name, 코멘트 멘션 시 표시. 내부/게스트 모두 편집 가능)
- 셀렉트: 역할
  - 내부 사용자: 기획자 / 개발자 / 디자이너
  - 게스트: "게스트" 고정 (disabled)
- 액션 항목: `[저장]` — `profiles.display_name` + `profiles.role` UPDATE

### 세션 관리

**elements**
- 액션 항목: `[로그아웃]` — 세션 종료 후 `/` 이동

## 상태

| 상태 | 표현 |
|------|------|
| 내부 사용자 | 역할 셀렉트 활성 (3개 선택지) |
| 게스트 | 역할 셀렉트 "게스트" 고정 + disabled |
| 로딩 중 | 스켈레톤 |
| 저장 중 | `[저장]` 비활성 + 스피너 |
| 저장 성공 | 토스트 메시지 + 시트 유지 |
| 저장 실패 | 에러 메시지 + 입력 유지 |

## 인수조건

- [ ] 내부 사용자: GitHub 아바타 + 사용자명이 표시된다
- [ ] 게스트: 빈 아바타 + 이메일이 표시된다
- [ ] 현재 선택된 역할이 기본 선택되어 있다
- [ ] 내부 사용자: 기획자/개발자/디자이너 3개 선택지로 변경 가능하다
- [ ] 게스트: 역할 셀렉트가 disabled 상태이다
- [ ] 이름 변경 + "저장" 클릭 시 `profiles.display_name`이 업데이트된다
- [ ] 역할 변경 + "저장" 클릭 시 `profiles.role`이 업데이트된다
- [ ] 저장 성공 시 토스트가 표시된다
- [ ] 변경 없이 "저장"을 눌러도 에러 없이 처리된다
- [ ] "로그아웃" 클릭 시 `supabase.auth.signOut()` 후 `/`로 이동한다
- [ ] `[X]` 또는 오버레이 클릭 시 시트가 닫힌다 (변경사항 미저장 시 확인 없이 닫힘)

## 비고

- 내부 사용자의 역할은 권한이 아닌 프로필 표시용 — 변경해도 접근 권한에 영향 없음
- 게스트의 역할(viewer)은 시스템이 부여, 변경 불가
- 추후 알림 설정, 테마 등 설정 항목 확장 가능
