# 상품 목록

## 개요

상품 검색 및 목록 조회 화면. 사용자가 카테고리별로 상품을 탐색하고, 검색/필터를 통해 원하는 상품을 빠르게 찾을 수 있다.

## 기능 정의

| 기능 ID | 기능명 | 설명 | 우선순위 |
|---------|--------|------|---------|
| SEARCH | 상품 검색 | 키워드 기반 검색. 최소 2글자 이상 입력 시 자동완성 | P0 |
| FILTER | 카테고리 필터 | 카테고리, 가격대, 브랜드별 필터링. 다중 선택 가능 | P0 |
| SORT | 정렬 | 최신순, 인기순, 가격 낮은순, 가격 높은순 | P1 |
| PAGINATION | 페이지네이션 | 무한 스크롤 방식. 한 번에 20개 로딩 | P1 |

## 화면 구성

### 상단 영역

- **검색바**: 전체 너비. placeholder "상품명, 브랜드명으로 검색"
- **필터 칩**: 검색바 아래. 활성 필터를 칩으로 표시, X 클릭으로 해제

### 상품 그리드

- **레이아웃**: PC 4열 / MO 2열 그리드
- **카드 구성**:
  - 상품 이미지 (1:1 비율)
  - 상품명 (최대 2줄, 말줄임)
  - 가격 (할인 시 원가 취소선 + 할인가 강조)
  - 리뷰 평점 + 리뷰 수

### 빈 상태

검색 결과가 없을 때:

> "검색 결과가 없습니다. 다른 키워드로 검색해보세요."

## 상태 정의

| 상태 | 조건 | UI 동작 |
|------|------|---------|
| 로딩 | 최초 진입 또는 필터 변경 시 | 스켈레톤 카드 8개 표시 |
| 정상 | 상품 1개 이상 | 그리드 렌더링 |
| 빈 결과 | 검색/필터 결과 0건 | 빈 상태 일러스트 + 메시지 |
| 에러 | API 실패 | "상품을 불러올 수 없습니다" + 재시도 버튼 |

## API 연동

```
GET /api/products
  ?keyword={검색어}
  &category={카테고리ID}
  &sort={정렬기준}
  &page={페이지번호}
  &limit=20
```

응답 예시:

```json
{
  "items": [
    {
      "id": "prod_001",
      "name": "프리미엄 무선 이어폰",
      "price": 89000,
      "discountPrice": 69000,
      "imageUrl": "/images/prod_001.jpg",
      "rating": 4.5,
      "reviewCount": 128
    }
  ],
  "total": 256,
  "hasMore": true
}
```

## 인터랙션

1. **검색**: 입력 후 300ms 디바운스 → API 호출
2. **필터**: 체크박스 선택 즉시 → 1페이지부터 재조회
3. **정렬**: 드롭다운 선택 즉시 → 1페이지부터 재조회
4. **무한 스크롤**: 하단 200px 도달 시 다음 페이지 로딩
5. **상품 클릭**: `/products/{id}` 상세 페이지로 이동
