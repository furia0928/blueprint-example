# 버전 발행

> 트리거: 화면 상세 헤더의 [발행] 버튼 클릭
> 상세: [features/VERSION.md](../../../features/VERSION.md)

## 화면 구성

### 현재 버전 표시

**elements**
- 읽기 전용 텍스트: "현재 최신 버전: {tag}" — 태그 없으면 "없음"

### 발행 유형 선택

**elements**
- 카드 2개 (라디오 선택):
  - 마이너: `v{현재}.+1` — "피드백 반영"
  - 메이저: `v{현재+1}.0` — "구조 변경"
- 첫 버전(기존 태그 없음): 카드 1개, `v1.0` 고정 (선택 불필요)
- 선택 변경 시 다음 버전 번호 미리보기가 갱신된다

### 미해결 코멘트 안내

**elements**
- 미해결(`status = open`) 있음: ⚠ 경고 — "미해결 코멘트 N개가 있습니다" + 이월 규칙 안내 ("'최신(미발행)'에 남아 다음 버전까지 이월됩니다")
- 미해결 없음: ✅ "모든 코멘트가 해결되었습니다"

### 액션

**elements**
- 액션 항목: `[발행]` 버튼 — 태그 생성 + 코멘트 귀속 처리
- `[X]` / 오버레이 클릭 → 다이얼로그 닫힘

## 상태

| 상태 | 표현 |
|------|------|
| 기본 | 현재 버전 + 발행 유형 카드 + 미해결 코멘트 안내 |
| 첫 버전 | 발행 유형 카드 1개(v1.0 고정) |
| 발행 중 | `[발행]` 비활성 + 스피너 |
| 발행 유형 미선택 + 제출 | "발행 유형을 선택해주세요" |
| 태그 중복 (동시 발행 경합) | "이미 발행된 버전입니다" |
| GitHub API 실패 | "버전 발행에 실패했습니다" + 재시도 |

## 인수조건

- [ ] 현재 최신 버전이 표시된다 (태그 없으면 "없음")
- [ ] 마이너(v1.1 → v1.2) / 메이저(v1.1 → v2.0) 중 선택한다
- [ ] 첫 버전인 경우 v1.0으로 고정 (선택 불필요)
- [ ] 미해결 코멘트가 있으면 경고가 표시되고, 발행은 차단하지 않는다
- [ ] "발행" 클릭 시 GitHub API로 최신 커밋에 태그를 생성한다
- [ ] 해결된 코멘트(`status = closed`)가 해당 버전에 귀속된다 (`comments.version` 업데이트)
- [ ] 미해결 코멘트(`status = open`)는 `latest`에 잔류한다
- [ ] 발행 완료 후 다이얼로그 닫힘 → 버전 드롭다운에 새 버전 반영
- [ ] 발행 중 로딩 상태가 표시된다 (버튼 비활성 + 스피너)

## 비고

- 동시 발행 경합: GitHub API가 중복 태그를 거부하므로 별도 락 불필요
- 태그는 기본 브랜치(main)의 HEAD 커밋에 생성
- 버전 발행은 App 토큰으로 수행 (시스템 행동)
- 발행 후 태그 캐시 무효화
