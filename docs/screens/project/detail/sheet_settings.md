# 프로젝트 설정

> 트리거: 화면 상세 [탑바](./area_top-bar.md)의 ⚙ 아이콘
> 타입: 시트 (shadcn Sheet, 우측 슬라이드)

## 화면 구성

세로 스크롤 시트. 위에서 아래로 **기본 정보 → 게스트 초대(오너) → 프로젝트 삭제(Admin)** 순서로 구성. 각 섹션은 권한에 따라 노출이 달라진다.

### [기본 정보](../../../features/PROJECT.md#info-프로젝트-기본-정보)

모든 멤버가 볼 수 있으며 이름만 편집 가능.

**elements**
- 텍스트 항목: PROJECT__INFO 프로젝트 이름 — 현재 이름 표시, 편집 가능
- 읽기 전용 항목: PROJECT__INFO GitHub 저장소 `owner/repo`
- 액션 항목: "저장" 버튼 — 변경 없으면 비활성

### [게스트 초대](../../../features/AUTH.md#guest_invite-게스트-초대-링크)

프로젝트 오너에게만 노출.

**elements**
- 섹션 제목: "게스트 초대"
- 액션 항목: "초대 링크 생성" 버튼
- 초대 링크 표시 영역:
  - 생성된 URL (읽기 전용)
  - 액션 항목: 📋 클립보드 복사
  - 만료일 표기 (기본 7일)
  - 만료된 경우 [↻] 재생성 버튼
- 승인 대기 목록 (AUTH__GUEST_APPROVAL `pending` 상태):
  - 이름 + 이메일
  - 액션 항목: ✓ 수락 / ✕ 거절
- 승인됨 목록 (AUTH__GUEST_APPROVAL `approved` 상태):
  - 이름 + 이메일
  - 액션 항목: "승인 해제"

### [프로젝트 삭제](../../../features/PROJECT.md#delete-프로젝트-삭제)

GitHub 저장소 Admin 권한자에게만 노출.

**elements**
- 액션 항목: "프로젝트 삭제" 버튼 (위험 스타일)
- 클릭 시 확인 다이얼로그: 프로젝트명 타이핑 확인 패턴

## 상태

| 상태 | 표현 |
|------|------|
| 시트 열림 (로딩) | 프로젝트 정보·게스트 목록 로딩 중 스켈레톤 |
| 일반 멤버 | 기본 정보만 노출 |
| 오너 | 기본 정보 + 게스트 초대 섹션 |
| Admin | 기본 정보 + (오너면 게스트) + 삭제 섹션 |
| 이름 빈값 | "프로젝트 이름을 입력해주세요" 유효성 |
| 저장 중 | "저장" 비활성 + 스피너 |
| 저장 성공 | 토스트 "저장되었습니다" + 시트 닫힘 |
| 저장 실패 | "설정 저장에 실패했습니다" + 재시도 |
| 초대 링크 생성 실패 | "초대 링크 생성에 실패했습니다" + 재시도 |
| 삭제 실패 | "프로젝트 삭제에 실패했습니다" + 재시도 |
| 미저장 상태에서 닫기 | 변경 사항 버려짐 (별도 경고 없음) |

## 인수조건

- [ ] 시트 열릴 때 프로젝트 정보와 게스트 목록을 로드해 표시한다
- [ ] 이름 변경 후 "저장" 클릭 시 `projects.name`이 업데이트되고 토스트가 표시되며 시트가 닫힌다
- [ ] 빈 이름은 저장되지 않는다
- [ ] GitHub 저장소명은 읽기 전용이다
- [ ] 게스트 초대 섹션은 프로젝트 오너(`projects.created_by`)에게만 노출된다
- [ ] "초대 링크 생성" 시 미만료 기존 링크가 있으면 재사용하고, 없으면 새로 생성한다
- [ ] 하나의 초대 링크를 여러 게스트에게 공유할 수 있다
- [ ] 만료된 링크는 [↻]로 새 토큰을 재생성할 수 있다
- [ ] 오너는 승인 대기 게스트를 수락(`approved`) 또는 거절(삭제)할 수 있다
- [ ] 오너는 승인된 게스트의 접근을 해제(`pending`으로 복귀)할 수 있다
- [ ] "프로젝트 삭제" 버튼은 저장소 Admin 권한자에게만 노출된다
- [ ] 삭제 확인 다이얼로그에서 프로젝트명 타이핑 확인을 통과해야 삭제된다
- [ ] 삭제 시 `projects`/`comments`/`github_cache`만 정리되고 GitHub 저장소·이슈는 유지된다
- [ ] 삭제 완료 후 `/projects`로 이동한다
- [ ] [X] 또는 오버레이 클릭 시 시트가 닫히며 미저장 변경은 버려진다

## 비고

- 게스트 데이터(pending/approved 목록)는 부모 컴포넌트(사이드바)에서 `getProjectGuests` Server Action으로 조회한 후 `guests` prop으로 전달한다. 시트 내부에서 useEffect로 데이터를 가져오지 않는다
- 게스트 목록 갱신은 `onRefreshGuests` 콜백으로 부모에 위임
- 초대 링크 만료 기본값은 7일
- GitHub App 설치 상태는 여기서 관리하지 않는다 (설치 해제 에러는 [INTEGRATION.md](../../../INTEGRATION.md) 참조)
- 게스트 초대·승인 관리 규칙 상세: [features/AUTH.md#guest_invite-게스트-초대-링크](../../../features/AUTH.md#guest_invite-게스트-초대-링크), [features/AUTH.md#guest_approval-오너-승인-관리](../../../features/AUTH.md#guest_approval-오너-승인-관리)
