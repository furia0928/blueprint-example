    파일이 선택되지 않은 초기 상태의 안내. 실 서비스에선 `/projects/[id]` 진입 시 첫 화면으로 자동 리다이렉트되므로 거의 노출되지 않는다. 탑바 우측 — 버전 드롭다운 앞에 배치 (DOM도 탑바).
    · `*.html` 파일 선택 시에만 노출 — `data-state-ref="WIREFRAME__RENDERING"`으로 뷰어 상태 참조
    · 클릭 시 [코멘트 시트](./sheet_comments.md) 토글
    · 시트 열림 시 기능 영역 하이라이트 활성화 (별도 와이어프레임으로 표현 예정)     `<iframe sandbox="allow-scripts" srcdoc="...">`로 격리 렌더링.
    · `allow-same-origin` 사용 금지 (sandbox 무력화 방지)
    · 브릿지 스크립트가 `</body>` 앞에 주입됨
    · iframe 높이는 `bp:resize` postMessage로 자동 동기화
    · 1MB 초과 파일은 Contents API `raw` 미디어 타입으로 요청 (최대 100MB)     iframe 위 absolute positioned 레이어에 렌더되는 코멘트 핀. 규칙은 [COMMENT.md#pin](../../../features/COMMENT.md#pin-핀-좌표-관리) 참조.
    · 좌표는 feature div 기준 비율(0~1) — 화면 크기 무관
    · 기본 `pointer-events: none` (iframe 내부 인터랙션 허용), 뱃지 자체만 `pointer-events: auto`
    · 뱃지 클릭 시 [코멘트 시트](./sheet_comments.md)에서 해당 코멘트 하이라이트
    · 변형: 미해결(default)·해결(secondary) 색상 구분     `*.md` 파일 선택 시 활성. react-markdown + remark-gfm로 마크다운 렌더링.
    · 테이블·코드 블록 등 GFM 문법 지원
    · 핀 오버레이 숨김 (코멘트는 .html 파일에만 가능 — 탑바 💬 버튼도 비활성)
    · 화면명세(`screen.md`, `area_*.md`)·기능명세(`features/*.md`)·플로우(`flows/*.md`)에 공통 사용     파일 fetch 중 뷰어 영역을 가리는 스켈레톤 플레이스홀더.
    · GitHub Contents API 응답 + 브릿지 스크립트 주입이 끝나기 전까지 노출
    · iframe 높이·rect 계산 완료 시 실제 뷰어로 교체     파일 fetch 실패 시 뷰어 자리에 표시되는 에러 안내.
    · 원인(네트워크 오류, 권한 박탈, 저장소 삭제 등)에 관계없이 동일 UI
    · "다시 시도" 버튼 — 클릭 시 현재 screenId·파일로 재fetch
    · 반복 실패 시 사용자에게 프로젝트 설정/권한 확인 유도 (추후)     프로젝트 저장소의 `docs/screens/` 아래 유효한 화면이 하나도 없을 때 노출.
    · 플러그인 사용법 링크로 유도 (외부 링크 — 새 탭)
    · 플러그인으로 화면 생성 후 push하면 실시간으로 파일 트리·뷰어 갱신됨 (새로고침 불필요)
  • origin 검증: `event.origin === 'null'` (sandbox iframe은 null origin)
