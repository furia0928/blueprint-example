# 파일 트리

> 소속: [screen.md](./screen.md)

## 유저스토리

- 사용자로서, 프로젝트의 전체 화면을 그룹별로 한눈에 보고 원하는 화면으로 빠르게 전환하고 싶다.
- 사용자로서, 화면이 많을 때 그룹을 접어서 필요한 그룹만 보고 싶다.
- 사용자로서, 목록에서 지금 보고 있는 화면이 어떤 것인지 바로 알고 싶다.

**elements**
- 그룹 헤더: 그룹명 + `▾/▸` 토글 (클릭 시 접기/펼치기)
- 화면 항목: 화면 폴더명 (클릭 시 화면 전환)
- 파일 타입 아이콘: `.html` → 오렌지, `.md` → 블루, 그 외 → 기본
- 현재 선택 표시: 활성 화면 하이라이트 (배경색/볼드)
- 스크롤: 목록이 길면 사이드바 내부 스크롤
- 너비: 240px 고정

## 인수조건

- [ ] GitHub `docs/screens/` 디렉토리 구조를 파싱하여 그룹/화면 트리가 표시된다
- [ ] 1-depth 폴더 = 그룹명 (한글 가능)
- [ ] 그룹 폴더 없이 바로 있는 화면 폴더는 "미분류" 그룹에 표시된다
- [ ] 그룹/화면 모두 알파벳순 정렬이다
- [ ] 화면 클릭 시 URL이 `/projects/[id]/screens/[screenId]`로 변경되고 중앙 뷰어가 갱신된다
- [ ] 그룹명(또는 ▾/▸ 아이콘) 클릭 시 하위 화면이 접기/펼치기 토글된다
- [ ] 현재 선택된 화면이 속한 그룹은 자동으로 펼쳐진다
- [ ] 접힌 상태는 세션 동안 유지된다
- [ ] 현재 선택된 화면이 시각적으로 구분된다 (배경색/볼드)
- [ ] URL의 screenId와 트리의 선택 상태가 동기화된다
- [ ] 딥링크 진입 시에도 해당 화면이 선택 상태로 표시된다
- [ ] `.html`과 `.md` 파일만 트리에 표시된다 (이미지·JSON 등 그 외 파일은 숨김)
- [ ] 그룹이면서 화면인 항목 클릭 시 화면으로 전환된다 (그룹 겸 화면 지원)
- [ ] 로딩 중에는 트리 스켈레톤이 표시된다

## 비고

- 화면 인식 규칙: `*screen*.md` + `*wireframe*.html` 한 쌍이 있어야 유효
- 검색 기능 미지원 — 그룹 접기/펼치기 + 알파벳순으로 탐색
- DB 테이블 없음 — GitHub 디렉토리에서 실시간 파싱 (캐싱 적용)
