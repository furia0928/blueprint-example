# 코멘트

> 트리거: 뷰어 툴바의 💬 코멘트 버튼 클릭 (HTML 파일 선택 시에만 노출)
> 상세: [features/COMMENT.md](../../../features/COMMENT.md) — 코멘트 전체, [features/VERSION.md](../../../features/VERSION.md) — 과거 버전 규칙

## 화면 구성

우측 슬라이드 시트 (360px 고정). 목록 뷰 ↔ 상세 뷰로 전환된다.

### 목록 뷰

**elements**
- 시트 헤더: "코멘트" 타이틀 + `[X]` 닫기 버튼
- 라벨 필터: shadcn Combobox. 다중 선택. 저장소 GitHub Labels
- 코멘트 카드 (정렬: 생성일 내림차순, 최신 위)
  - 제목, 라벨 뱃지, assignee, 작성 시각
  - 클릭 가능 → 상세 뷰 전환
  - hover 시 뷰어의 해당 핀 강조
  - featureId가 현재 와이어프레임에 없으면 "⚠ 삭제된 기능" 배지

### 상세 뷰

**elements**
- `[← 목록]` 복귀 버튼
- 이슈 본문 영역: 제목, 라벨, assignee, 본문, 작성자, 작성일
- 액션 버튼:
  - `[수정]` — 본인 코멘트에만 표시. 클릭 시 [코멘트 작성/수정 다이얼로그](./dialog_comment.md) 수정 모드 열림
  - `[해결]` — 확인 후 Issue close + `comments` row 삭제
- 댓글 스레드: 시간순 표시
- 댓글 입력: 텍스트 입력 + 전송 버튼. @멘션 자동완성 (저장소 collaborator)

## 상태

| 상태 | 표현 |
|------|------|
| 코멘트 있음 | 목록 뷰 (카드 리스트) |
| 코멘트 없음 | "아직 코멘트가 없습니다. 와이어프레임 위에서 기능 영역을 클릭하여 첫 코멘트를 남겨보세요" |
| 상세 로딩 중 | 상세 뷰 스켈레톤 |
| 과거 버전 | 목록 표시 + "이 버전에서는 새 코멘트를 생성할 수 없습니다" 안내 |
| 딥링크 진입(`?comment=123`) | 자동으로 해당 코멘트 상세 뷰 열림 |

## 인수조건

### 목록

- [ ] 현재 화면(screenId) + 버전의 코멘트가 `comments` 테이블에서 조회된다
- [ ] 각 카드에 제목·라벨·assignee·작성 시각이 표시된다
- [ ] 생성일 내림차순으로 정렬된다
- [ ] 이월된 미해결 코멘트도 함께 표시된다
- [ ] featureId가 현재 와이어프레임에 없는 코멘트는 "삭제된 기능"으로 표시된다
- [ ] PC/MO 분리 모드에서는 현재 뷰포트(파일)의 코멘트만 표시된다
- [ ] 코멘트 hover 시 뷰어에서 해당 핀이 강조된다
- [ ] Supabase Realtime으로 INSERT/UPDATE가 실시간 반영된다

### 라벨 필터

- [ ] 라벨 필터가 combobox(다중 선택)로 제공된다
- [ ] 저장소의 GitHub Labels 목록이 선택지로 표시된다
- [ ] 선택한 라벨에 해당하는 코멘트만 필터링되고, 뷰어 핀도 연동된다
- [ ] 필터 해제 시 전체 코멘트 표시

### 상세

- [ ] 코멘트 클릭 시 상세 뷰로 전환된다
- [ ] GitHub Issues API로 본문과 댓글을 fetch한다
- [ ] 제목·라벨·assignee·본문·작성자·작성일이 표시된다
- [ ] `[← 목록]` 클릭 시 목록 뷰로 복귀한다
- [ ] `?comment=123` 딥링크 시 자동으로 해당 상세 뷰가 열린다

### 수정/해결

- [ ] 본인이 작성한 코멘트의 상세 뷰에만 `[수정]` 버튼이 표시된다
- [ ] `[수정]` 클릭 시 코멘트 작성/수정 다이얼로그가 수정 모드로 열리고, 기존 필드가 채워진다
- [ ] `[해결]` 클릭 시 확인 다이얼로그가 표시된다
- [ ] 확인 시 GitHub Issue close + `comments` row 삭제
- [ ] 해결 후 목록 뷰로 복귀하고 해당 카드·핀이 제거된다

### 댓글

- [ ] 상세 뷰 하단에 댓글 목록이 시간순으로 표시된다
- [ ] 댓글 입력 필드 + 전송 버튼이 있다
- [ ] 댓글 전송 시 GitHub Issue comment API로 생성된다 (사용자 OAuth 토큰)
- [ ] `@멘션` 입력 시 저장소 collaborator 목록에서 자동완성된다
- [ ] 과거 버전에서도 댓글 추가가 가능하다

## 비고

- 코멘트 목록은 `comments` 테이블에서 조회 (페이지네이션 없음 — 버전별 코멘트 수가 적음)
- 상세 내용(본문·댓글)은 클릭 시 GitHub Issues API로 on-demand fetch
- 과거 버전: 새 핀 생성 불가, 댓글 추가 + close는 가능
- `@멘션` 알림은 GitHub 자체 알림에 위임
- 라벨 관리는 저장소에서 직접 (플랫폼에서 별도 관리하지 않음)
