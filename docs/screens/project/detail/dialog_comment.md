# 코멘트 작성/수정

> 트리거: (작성) 코멘트 시트 열린 상태에서 뷰어의 기능 영역 클릭, (수정) 코멘트 상세에서 [수정] 클릭
> 상세: [features/COMMENT.md](../../../features/COMMENT.md)

## 화면 구성

단일 다이얼로그. 작성/수정 모드에 따라 제목·버튼·초기값만 다르고 폼 인터페이스는 동일.

### 위치 표시 (읽기 전용)

**elements**
- 핀 정보: `📌 {featureId} 영역 ({x}, {y})` — feature div 기준 비율(0~1) 좌표
- 작성·수정 모두 읽기 전용. 위치 변경은 지원하지 않음 (삭제 후 재작성)

### 입력 폼

**elements**
- 텍스트 항목: 제목 (필수)
- 텍스트 영역: 본문 (선택)
- 태그 선택: 라벨 (combobox, 저장소 GitHub Labels에서 다중 선택)
- 태그 선택: 담당자 (저장소 collaborator에서 다중 선택)

### 액션

**elements**
- 작성 모드: `[작성]` 버튼 단일
- 수정 모드: `[취소] [저장]` 버튼
- `[X]` / 오버레이 클릭 → 다이얼로그 닫힘 (변경사항 버림)

## 상태

| 상태 | 표현 |
|------|------|
| 작성 모드 | 제목 "코멘트 작성", 빈 폼, `[작성]` 버튼 |
| 수정 모드 | 제목 "코멘트 수정", 기존 데이터 채워짐, `[취소] [저장]` 버튼 |
| 제목 미입력 + 제출 | "제목을 입력해주세요" 유효성 표시 |
| 저장 중 | 버튼 비활성 + 스피너 |
| Issue API 실패 | "코멘트 저장에 실패했습니다" + 재시도. 입력 유지 (다이얼로그 안 닫힘) |
| Supabase 저장 실패(작성) | 자동 재시도 3회 → 실패 시 "GitHub에는 생성됐지만 플랫폼에 반영되지 않았습니다" |
| 라벨/collaborator 로딩 실패 | 해당 필드 비활성 + "불러올 수 없습니다" |

## 인수조건

- [ ] 작성 모드: 클릭한 featureId와 좌표(x, y)가 다이얼로그 상단에 표시된다
- [ ] 작성 모드: 모든 필드가 빈 상태로 열린다
- [ ] 수정 모드: 기존 제목·본문·라벨·담당자가 폼에 채워진 상태로 열린다
- [ ] 수정 모드: 핀 위치는 읽기 전용이다
- [ ] 본인이 작성한 코멘트만 수정 가능하다 (타인 코멘트에는 [수정] 버튼 미표시)
- [ ] "작성" 클릭 시: GitHub Issue 생성(본문에 `<!-- blueprint-meta -->` 포함) → issue_number를 받아 `comments` 테이블 저장
- [ ] "저장" 클릭 시: Issue PATCH + `comments` 테이블 UPDATE
- [ ] 생성/수정 완료 후 다이얼로그 닫힘 → 코멘트 목록 + 뷰어 핀 즉시 반영
- [ ] 좌표는 feature div 기준 비율(0~1)로 저장된다
- [ ] Webhook `issues` 이벤트로 누락된 코멘트가 자동 보정된다

## 비고

- 과거 버전에서는 작성 모드로 열리지 않음 (기능 영역 클릭 비활성). 수정 모드는 과거 버전에서도 가능
- 핀 위치 변경은 미지원 — 삭제 후 재작성
