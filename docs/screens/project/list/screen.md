# 프로젝트 목록

> 라우트: `/projects`
> 인증: 필수 (비로그인 시 `/login`으로 리다이렉트)

## 화면 구성

상단은 헤더(로고·알림·프로필 아바타), 본문은 페이지 타이틀 + "새 프로젝트" 액션 + 프로젝트 카드 그리드.

### [프로젝트 카드 그리드](../../../features/PROJECT.md)

**elements**
- 페이지 타이틀: "프로젝트"
- 액션 항목: "새 프로젝트" 버튼 — [프로젝트 생성 다이얼로그](./dialog_create.md) 트리거
- 프로젝트 카드 (반복):
  - 즐겨찾기 토글 (별 아이콘)
  - PROJECT__INFO: 프로젝트명, GitHub 저장소 `owner/repo`
  - PROJECT__STATS: 화면 수, 미해결 코멘트 수
  - 예제 프로젝트는 "예제" 뱃지 표시
- 정렬: 즐겨찾기 우선 → 생성일 내림차순

### 헤더

**elements**
- 로고: Blueprint (클릭 시 `/projects` 복귀)
- 알림 아이콘 (추후)
- 프로필 아바타 — 클릭 시 [프로필 설정 시트](../../common/sheet_profile.md)

## 상태

| 상태 | 표현 |
|------|------|
| 로딩 중 | 카드 스켈레톤 |
| 프로젝트 있음 | 카드 그리드 (즐겨찾기 우선 → 생성일 내림차순) |
| 프로젝트 없음 | 중앙 빈 상태 카드: "아직 프로젝트가 없습니다 / 예제 프로젝트로 체험해보세요" + "예제 보기" + "새 프로젝트" 버튼 |
| GitHub API 에러 | "저장소 정보를 불러올 수 없습니다" + 재시도 버튼 |
| 저장소 삭제된 프로젝트 | 카드에 에러 표시 (진입 불가) |

## 인수조건

- [ ] GitHub collaborator인 저장소와 매칭되는 프로젝트만 노출된다 (public이어도 collaborator 아니면 미표시)
- [ ] 즐겨찾기 클릭 시 `project_favorites` 테이블에 INSERT/DELETE되며 상단 정렬된다
- [ ] 즐겨찾기 상태는 사용자별이며 다른 사용자에게 영향이 없다
- [ ] 프로젝트 카드 클릭 시 `/projects/[id]`로 이동하고, 첫 번째 화면으로 리다이렉트된다
- [ ] 화면이 없는 프로젝트는 빈 뷰어 상태로 진입한다
- [ ] 프로젝트 0개일 때 예제 안내와 "새 프로젝트" 버튼이 함께 표시된다
- [ ] "예제 보기" 클릭 시 예제 저장소(`blueprint-emotion/blueprint-example`)를 사용자 계정에 fork 후 DB 프로젝트 생성 → 목록에 노출된다
- [ ] 헤더 프로필 아바타 클릭 시 프로필 설정 시트가 열린다

## 진입 경로

| 출발 | 트리거 |
|------|------|
| 로그인 완료 | `/projects`로 자동 이동 |
| 로그인 상태에서 `/` 접근 | `/projects`로 리다이렉트 |
| 헤더 로고 클릭 | 어느 페이지에서든 복귀 |
| 프로젝트 생성 완료 | 다이얼로그 닫힘 + 새 프로젝트 카드 노출 |

## 비고

- 멤버 정보는 DB에 저장하지 않고 매번 GitHub API로 접근 가능 저장소를 조회해 매칭한다
- 화면 수는 GitHub 디렉토리 구조 파싱 (캐싱 적용)
- 미해결 코멘트 수는 `comments` 테이블 `status = 'open'` 카운트

### 예제 프로젝트

- 예제 저장소: `blueprint-emotion/blueprint-example` (org 소유, 샘플 와이어프레임 HTML + 화면명세 + 버전 태그 포함)
- 흐름: "예제 보기" → 사용자 OAuth 토큰으로 fork (`POST /repos/{owner}/{repo}/forks`) → fork된 저장소로 DB 프로젝트 생성 → 목록 노출
- fork 기반이라 코멘트(Issue)가 사용자별 완전 격리
- 별도 거절 플래그 없음 — 프로젝트 0개이면 항상 예제 안내 표시, fork하면 자연스럽게 일반 목록으로 전환
