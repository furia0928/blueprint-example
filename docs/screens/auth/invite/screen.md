# 게스트 초대

> 라우트: `/invite/[token]`
> 인증: 불필요 (페이지 진입 후 단계적으로 인증 유도)
> 상세 플로우: [flows/AUTH.md#게스트-초대--게스트-측](../../../flows/AUTH.md#게스트-초대--게스트-측)

## 화면 구성

중앙 카드 1개. 토큰 + 사용자 인증 + 프로필 + 멤버십 4축의 조합으로 영역이 모드 전환된다 (분기는 `## 상태` 표 참조).

### [신원 확인](../../../features/AUTH.md#guest_register-게스트-등록)

미로그인 게스트가 자기 이메일을 식별하고 인증 방식이 결정되는 영역.

**elements**
- 헤더: "[프로젝트명]에 초대되었습니다" — `invitations.project_id` → `projects.name` 조회
- 텍스트 항목: 이메일 — 신규 분기 시 활성, 기존 계정 분기 시 disabled
- 액션 항목: "다음" — `checkGuestEmail`로 기존 계정 여부 판별
- 텍스트 항목: 비밀번호 — 기존 계정 분기에서만 표시, 마스킹
- 액션 항목: "로그인" — `signInWithPassword` 호출 (기존 계정)
- 매직링크 발송 안내: 신규 계정 시 `signInWithOtp` 후 "이메일을 확인해주세요"
- 액션 항목: "다른 이메일로 시도" — 이메일 입력 단계로 복귀

### [게스트 등록](../../../features/AUTH.md#guest_register-게스트-등록)

로그인 직후 `display_name`이 없는 사용자의 프로필 초기 설정 영역.

**elements**
- 헤더: "환영합니다! 계정 정보를 입력해주세요"
- 텍스트 항목: 이름 (`display_name`)
- 텍스트 항목: 비밀번호 + 비밀번호 확인 (마스킹). 안내 문구: "다음에 이메일과 비밀번호로 로그인할 수 있습니다"
- 액션 항목: "시작하기" — `registerGuest` 호출

비밀번호 정책: 8자 이상 + 대문자 + 숫자 + 특수문자.

### [승인 대기](../../../features/AUTH.md#guest_approval-오너-승인-관리)

`invitation_members.status = pending` 상태의 안내 영역. 조작 불가.

**elements**
- 시계 아이콘
- 안내 문구: "승인 대기 중입니다. 프로젝트 오너가 접근을 승인하면 이용할 수 있습니다"
- 보조 안내: "승인 후 다시 접속해주세요"

## 상태

| 조건 | 화면 |
|------|------|
| 토큰 무효 | 404 |
| 토큰 만료 | "만료된 링크입니다 / 관리자에게 새 링크를 요청하세요" |
| 토큰 유효 + 미로그인 | 신원 확인 (이메일 입력) |
| 토큰 유효 + 미로그인 + 기존 계정 | 신원 확인 (비밀번호 로그인) |
| 토큰 유효 + 미로그인 + 신규 + 매직링크 발송됨 | 신원 확인 (이메일 확인 안내) |
| 토큰 유효 + 로그인 + `display_name` 없음 | 게스트 등록 |
| 토큰 유효 + 로그인 + `pending` | 승인 대기 |
| 토큰 유효 + 로그인 + `approved` | `/projects/[id]`로 즉시 리다이렉트 |
| 입력 검증 실패 | 필드별 안내 (이메일/비밀번호 정책 위반/불일치 등) |
| 매직링크 발송 실패 | "이메일 발송에 실패했습니다. 다시 시도해주세요" |
| 비밀번호 로그인 실패 | "비밀번호가 올바르지 않습니다" |
| 프로젝트 오너가 게스트 등록 시도 | "프로젝트 오너는 게스트로 등록할 수 없습니다" |

## 인수조건

- [ ] 토큰 무효 시 404, 만료 시 만료 안내 화면을 표시한다
- [ ] [다음] 클릭 시 `checkGuestEmail`로 기존 계정 여부를 판별해 비밀번호 로그인 또는 매직링크 발송으로 분기한다
- [ ] 매직링크 콜백은 `/api/auth/callback?next=/invite/[token]` 으로 돌아와 페이지 상태가 갱신된다
- [ ] 비밀번호는 정책(8자 이상 + 대문자 + 숫자 + 특수문자)을 위반하면 필드별 메시지를 표시한다
- [ ] 비밀번호와 확인이 일치하지 않으면 "비밀번호가 일치하지 않습니다"
- [ ] 프로젝트 오너 본인이 자기 프로젝트로 게스트 등록 시도하면 거부한다 (`registerGuest`)
- [ ] 등록 완료 후 `invitation_members`는 `pending`으로 upsert된다 (중복 없음)
- [ ] `approved` 상태로 진입하면 즉시 `/projects/[id]`로 리다이렉트한다
- [ ] 미인증 상태 접근을 허용하기 위해 미들웨어 인증 체크에서 제외된다

## 진입 경로

| 출발 | 트리거 |
|------|------|
| 오너가 공유한 초대 링크 클릭 | URL의 `[token]` 검증 |
| 매직링크 이메일 클릭 | `/api/auth/callback?next=/invite/[token]` 경유 |
