# 비밀번호 재설정

> 라우트: `/reset-password`
> 인증: 불필요 (Supabase 매직링크가 URL fragment에 세션 토큰 포함)
> 진입점: 비밀번호 재설정 이메일 내 링크 클릭
> 상세 플로우: [flows/AUTH.md#비밀번호-재설정](../../../flows/AUTH.md#비밀번호-재설정)

## 화면 구성

중앙 카드 1개. 세션 유효성에 따라 입력 / 완료 / 만료 안내로 모드 전환.

### [새 비밀번호 입력](../../../features/AUTH.md#password_reset-비밀번호-재설정)

`PASSWORD_RECOVERY` 세션이 감지된 경우의 폼 영역.

**elements**
- 카드 제목: "새 비밀번호 설정"
- 텍스트 항목: 새 비밀번호 (마스킹). 안내 문구: "8자 이상, 대문자, 숫자, 특수문자 포함"
- 텍스트 항목: 비밀번호 확인 (마스킹) — 새 비밀번호와 일치 검증
- 액션 항목: "비밀번호 변경" 버튼 — `supabase.auth.updateUser({ password })` 호출

비밀번호 정책: 8자 이상 + 대문자 + 숫자 + 특수문자 (`registerGuest`와 동일).

## 상태

| 상태 | 표현 |
|------|------|
| 세션 유효 + 입력 전 | 새 비밀번호 폼 표시 |
| 입력 중 정책 위반 | 필드 아래 빨간 텍스트로 미충족 항목 표시 |
| 비밀번호 불일치 | "비밀번호가 일치하지 않습니다" |
| 변경 중 | 버튼 로딩 표시 |
| 변경 성공 | "✓ 비밀번호가 변경되었습니다 / 새 비밀번호로 로그인하세요" + "로그인하러 가기" 버튼 |
| 변경 실패 | "비밀번호 변경에 실패했습니다. 다시 시도해주세요" |
| 세션 만료 / 토큰 없음 (직접 접근 등) | "링크가 만료되었거나 유효하지 않습니다" + "로그인으로" 버튼 |

## 인수조건

- [ ] `onAuthStateChange`로 `PASSWORD_RECOVERY` 이벤트가 감지된 경우에만 비밀번호 입력 폼을 표시한다
- [ ] 세션이 없으면 만료 안내와 "로그인으로" 버튼을 표시한다
- [ ] 비밀번호 정책 위반 시 필드별 메시지를 표시한다
- [ ] 새 비밀번호와 확인이 일치하지 않으면 "비밀번호가 일치하지 않습니다"
- [ ] 변경 성공 후 자동 로그인하지 않고 `/login`으로 안내한다 (recovery 세션 제약)
- [ ] Supabase Redirect URLs에 `/reset-password` 패턴이 등록되어 있어야 한다

## 진입 경로

| 출발 | 트리거 |
|------|------|
| 비밀번호 재설정 이메일 링크 | `{site_url}/reset-password#access_token=...&type=recovery` |
| 직접 URL 입력 | 세션 없이 진입 → 만료 안내 |

## 비고

- 재설정 이메일 발송은 [로그인 페이지](../login/screen.md)의 "비밀번호 찾기" 모드에서 처리
- 비밀번호 정책은 게스트 등록([invite](../invite/screen.md))과 동일하게 유지
