# 역할 선택

> 라우트: `/onboarding`
> 인증: 필수 (비로그인 시 `/login`으로 리다이렉트)
> 상세 플로우: [flows/AUTH.md#내부-사용자-로그인-github-oauth](../../../flows/AUTH.md#내부-사용자-로그인-github-oauth)

## 화면 구성

중앙 카드 1개. 환영 메시지 + 이름 입력 + 역할 카드 3개 + "시작하기" 버튼.

### [프로필 초기 설정](../../../features/AUTH.md#oauth_login-github-oauth-로그인)

**elements**
- 환영 메시지: "환영합니다! 프로필을 설정해주세요"
- 텍스트 항목: 이름 (`display_name`) — 코멘트 멘션 시 표시될 이름임을 안내
- 라디오 카드 그룹: 기획자 / 개발자 / 디자이너 (각각 아이콘 + 한 줄 설명)
- 액션 항목: "시작하기" 버튼 — 이름·역할 모두 입력 시 활성화
- 보조 안내 문구: "나중에 변경할 수 있습니다"

> 게스트(뷰어)는 이 화면을 거치지 않는다. 초대 플로우([screens/invite](../invite/screen.md))에서 이름만 입력하고 `role: viewer`로 자동 부여된다.

## 상태

| 상태 | 표현 |
|------|------|
| 기본 | 환영 메시지 + 빈 입력 + 비활성 시작하기 |
| 입력 완료 | 시작하기 버튼 활성화 |
| 저장 중 | 시작하기 버튼 로딩 표시 |
| 미입력 시도 | 미입력 항목 안내 표시 |

## 인수조건

- [ ] 이름 또는 역할이 미입력이면 "시작하기"가 비활성화된다
- [ ] 저장 완료 후 `/projects`로 이동한다 (또는 로그인 시 `next` 파라미터 경로)
- [ ] `profiles.role` 또는 `profiles.display_name`이 null인 사용자가 보호 경로에 접근하면 미들웨어가 `/onboarding`으로 리다이렉트한다
- [ ] 이미 역할이 있는 사용자가 `/onboarding` 직접 접근 시 `/projects`로 리다이렉트한다
- [ ] 비로그인 상태에서 접근 시 `/login`으로 리다이렉트한다
- [ ] 브라우저 뒤로가기로 이탈해도 다음 보호 경로 진입 시 다시 리다이렉트된다

## 진입 경로

| 출발 | 트리거 |
|------|------|
| 로그인 직후 (프로필 미완성) | 콜백에서 자동 진입 |
| 보호 경로 접근 시 | 미들웨어가 `/onboarding`으로 리다이렉트 |

## 비고

- 내부 사용자의 역할은 권한 제어가 아닌 프로필 표시용. 실제 권한은 GitHub collaborator가 담당
- [프로필 시트](../../common/sheet_profile.md)에서 이름/역할 변경 가능
