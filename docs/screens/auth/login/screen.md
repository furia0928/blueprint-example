# 로그인

> 라우트: `/login`
> 인증: 불필요 (로그인 상태면 `/projects`로 리다이렉트)
> 상세 플로우: [features/AUTH.md](../../../features/AUTH.md)

## 화면 구성

중앙 카드 1개. 모드 전환으로 GitHub 로그인 / 게스트 폼 / 비밀번호 재설정을 표시한다.

### [GitHub 로그인](../../../features/AUTH.md#oauth_login-github-oauth-로그인)

**elements**
- 액션 항목: GitHub OAuth 인증을 트리거하는 단일 버튼 (기본 CTA)
- 안내 문구: "GitHub 계정으로 Blueprint Platform에 접속합니다"

### [게스트 로그인](../../../features/AUTH.md#password_login-게스트-비밀번호-로그인)

**elements**
- 토글 액션: "게스트 로그인" 버튼 (`variant=outline`) — 클릭 시 폼 펼침/접힘
- 텍스트 항목: 이메일, 비밀번호 (마스킹 + 보기 토글)
- 액션 항목: "비밀번호 찾기" 링크 (폼 우측 하단) — 클릭 시 재설정 모드 전환
- 액션 항목: "로그인" 버튼 (`signInWithPassword` 호출)
- 안내 문구: "초대받은 게스트는 이메일로 로그인하세요" (토글 버튼 아래)

### [비밀번호 재설정](../../../features/AUTH.md#password_reset-비밀번호-재설정)

**elements**
- 모드 전환으로 카드 전체를 교체 (별도 라우트 없음)
- 카드 제목: "비밀번호 재설정"
- 텍스트 항목: 이메일
- 액션 항목: "재설정 링크 발송" 버튼 (`resetPasswordForEmail` 호출)
- 발송 완료 후: "다시 보내기" 액션 항목
- 복귀 액션: "← 로그인으로" 링크

## 상태

| 상태 | 표현 |
|------|------|
| 기본 | GitHub 로그인 카드 + 게스트 토글 접힘 |
| 게스트 폼 펼침 | 이메일/비밀번호 입력 영역 표시 |
| OAuth 진행 중 | 버튼 비활성화 |
| 재설정 모드 | 카드 제목 변경, 비밀번호 필드 숨김, "재설정 링크 발송" 버튼 |
| 발송 완료 | 안내 메시지 + "다시 보내기" 표시 |
| OAuth 취소 | 카드 상단 "로그인이 취소되었습니다" + 재시도 버튼 |
| 게스트 자격 증명 실패 | "이메일 또는 비밀번호가 올바르지 않습니다" |
| 네트워크 에러 | "네트워크 연결을 확인해주세요" + 재시도 버튼 |
| 재설정 발송 실패 | "이메일 발송에 실패했습니다. 다시 시도해주세요" |

## 인수조건

- [ ] 로그인 상태에서 `/login` 접근 시 `/projects`로 깜빡임 없이 리다이렉트한다
- [ ] 비로그인 + 보호 경로 직접 접근 시 `/login?next=원래경로` 로 이동하고, 로그인 후 `next` 경로로 복귀한다
- [ ] OAuth 취소 시 재시도 버튼이 표시되고, 클릭 시 OAuth 플로우 처음부터 재시작
- [ ] `provider_token` 저장 실패 시 자동 재시도 3회, 실패 시 `prompt: 'consent'`로 강제 재인증
- [ ] 게스트 로그인 토글은 클릭 시 폼이 즉시 펼쳐지고, 다시 클릭하면 접힌다
- [ ] 비밀번호 재설정 모드 전환은 페이지 이동 없이 카드 내 상태 전환으로 처리한다
- [ ] 비밀번호 변경 완료 후 자동 로그인하지 않고 `/login`으로 안내한다 (recovery 세션 제약)

## 진입 경로

| 출발 | 트리거 |
|------|------|
| 비로그인 + 보호 경로 직접 접근 | 미들웨어 리다이렉트 (`/login?next=...`) |
| 로그아웃 | 헤더 메뉴 → "로그아웃" |
| 게스트 초대 링크 클릭 (기존 계정) | 이메일 입력 → 기존 계정 감지 → 비밀번호 로그인 화면으로 |
