# 랜딩

> 라우트: `/`
> 인증: 불필요 (로그인 상태면 `/projects`로 서버 사이드 리다이렉트)

## 화면 구성

수직 스크롤 레이아웃. 상단에서 하단까지 6개 섹션.

### 헤더

**elements**
- 로고: "Blueprint"
- 액션 항목: "GitHub 로그인" 버튼 (우측) → `/login`

### 히어로

**elements**
- 제목: 서비스 핵심 가치를 전달하는 한 줄 태그라인
- 부제: 대상 사용자 설명 ("기획자·개발자·디자이너가 함께 보는 협업 플랫폼")
- 액션 항목: "GitHub로 시작하기" CTA 버튼 (기본) → `/login`
- 이미지: 플랫폼 실제 모습 목업 (핀 코멘트 찍힌 뷰어)

### 기능 소개

**elements**
- 카드 3개 그리드:
  - 핀 코멘트: 와이어프레임 위에 직접 핀을 찍어 피드백
  - 화면명세: 기획 의도를 메타데이터로 구조화하여 공유
  - 버전 관리: Git 태그로 버전을 발행하고 이력 추적

### 사용 흐름

**elements**
- 3단계 플로우 시각화: 와이어프레임 생성 → GitHub push → 팀원 피드백

### 하단 CTA

**elements**
- 액션 항목: "GitHub로 시작하기" 버튼 (히어로와 동일) → `/login`

### 푸터

**elements**
- 저작권 표시

## 상태

| 상태 | 표현 |
|------|------|
| 비로그인 | 기본 랜딩 표시 |
| 로그인 상태 | `/projects`로 리다이렉트 (이 페이지 미노출) |

## 인수조건

- [ ] 히어로에 태그라인, 부제, CTA 버튼, 목업 이미지가 모두 포함된다
- [ ] 핵심 기능 3개가 카드 그리드로 요약된다
- [ ] 사용 흐름이 3단계로 시각화된다
- [ ] 하단에 CTA가 반복되어 스크롤 후에도 접근 가능하다
- [ ] 모든 CTA 클릭 시 `/login`으로 이동한다
- [ ] 로그인 상태에서 `/` 접근 시 `/projects`로 깜빡임 없이 서버 사이드 리다이렉트한다
- [ ] OG 태그(서비스 설명, 미리보기 이미지)가 설정되어 있다
