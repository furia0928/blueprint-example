# 용어사전

이 프로젝트의 모든 문서(PRD·features·screens·flows·INTEGRATION 등)에서 사용하는 용어의 **공식 표기**와 **의미**를 정의한다. 새 문서 작성 시 이 사전의 용어를 사용하고, 변종이 발견되면 이 사전 쪽으로 통일한다.

> 이 문서는 용어 SSOT다. 도메인 규칙의 SSOT는 각 `features/*.md`, 스키마 SSOT는 `DB.md`.

## 문서 종류

| 공식 용어 | 정의 | 금지 변종 |
|-----------|------|-----------|
| **화면명세** | 화면 단위 요구사항 문서. `docs/screens/**/*.md` (screen·sheet·dialog·area) | "기획서", "화면 기획서" |
| **기능명세** | 도메인 단위 비즈니스 규칙 문서. `docs/features/*.md` | "기획서", "기능 기획서" |
| **플로우** | 화면·기능을 가로지르는 journey 다이어그램. `docs/flows/*.md` | "플로우차트", "흐름도" |
| **와이어프레임** | 기획자가 플러그인으로 생성하는 HTML 목업. `*.html` | "와프", "시안" |

> PRD 등에서 "기획서(MD)"로 기술된 부분은 **화면명세+기능명세**를 통칭한 구어다. 공식 문서에서는 개별 명칭으로 쓴다.

## 사용자·역할

| 공식 용어 | 정의 | 비고 |
|-----------|------|------|
| **내부 사용자** | GitHub OAuth로 로그인하는 팀원 | 역할 셀렉트에서 기획자/개발자/디자이너 중 선택 |
| **기획자** | role 값 `planner` | UI 표시: "기획자" |
| **개발자** | role 값 `developer` | UI 표시: "개발자" |
| **디자이너** | role 값 `designer` | UI 표시: "디자이너" |
| **게스트** | 외부 초대된 사용자. role 값 `viewer` | **UI 표시명은 "게스트"**, DB 값은 `viewer`. 혼동 방지를 위해 문서에서는 "게스트(role=viewer)" 병기 |
| **프로젝트 오너** | `projects.created_by` 사용자. 게스트 초대·승인 권한 보유 | "프로젝트 관리자"라는 표현은 금지 (Admin과 혼동) |
| **Admin** | GitHub 저장소 Admin 권한자. 프로젝트 삭제 권한 보유 | GitHub 개념을 그대로 사용. "저장소 관리자"는 같은 의미이나 공식 표기는 "Admin" |
| **Collaborator** | GitHub 저장소 collaborator. 프로젝트 접근 권한의 기준 | 원문 그대로 사용 |

**중요 구분**: 오너와 Admin은 **다른 개념**이다.
- 오너 = DB `projects.created_by` → 게스트 초대·승인 권한
- Admin = GitHub 저장소 Admin → 프로젝트 삭제 권한
- 두 권한은 독립적이며 한 사람이 둘 다일 수도, 한쪽만일 수도 있다.

**금지 변종**:
- "고객" → "게스트"로 통일. "고객"은 초기 문서에서 구어로 쓰였으나 공식 표기가 아니다.
- "뷰어(역할)"와 "뷰어(UI 컴포넌트)" 혼동 주의 — 아래 UI 용어 참조.

## 코멘트 체계

| 공식 용어 | 정의 | 비고 |
|-----------|------|------|
| **코멘트** | 와이어프레임 위 특정 기능 영역에 찍는 피드백. GitHub Issue 1건에 대응. `comments` 테이블 row 1개 | Issue 본문 + 메타데이터(좌표·screenId·version) 포함 |
| **댓글** | 코멘트(=Issue) 아래의 스레드. GitHub Issue comments에 대응 | 플랫폼 DB에 저장 안 함. GitHub에 위임 |
| **핀** | 코멘트의 시각적 마커 (뷰어 오버레이의 뱃지) | |
| **핀 코멘트** | 제품 기능 이름 — "와이어프레임 위 클릭으로 생성되는 코멘트" 기능 전체 | UI 문구·마케팅 용어 |
| **게스트 코멘트** | 게스트가 작성해 GitHub App(봇)이 대리 생성하는 코멘트 | `guest` 라벨 자동 부착 |

**중요**: "코멘트"와 "댓글"은 **다른 것**이다. 코멘트 = 최상위 Issue 단위, 댓글 = 스레드 답글. 혼용 금지.

## 와이어프레임·UI 구조

| 공식 용어 | 정의 | 비고 |
|-----------|------|------|
| **뷰어(영역)** | 화면 상세 중앙의 와이어프레임 렌더 영역. [area_viewer.md](./screens/project/detail/area_viewer.md) | "와이어프레임 뷰어"로 풀어 써도 됨 |
| **파일 트리** | 좌측 사이드바의 그룹/화면 트리 | "사이드바"와 같은 의미지만 "파일 트리"가 공식 |
| **탑바** | 상단 헤더 (브레드크럼·버전·발행·설정) | |
| **기능 영역** | 와이어프레임 HTML의 클릭 가능한 기능 구획 | 구현 용어 `feature div`, 컴포넌트 이름 `bp-section`과 같은 대상. 문서에서는 **"기능 영역"**으로 통일 |
| **섹션** | `bp-section` 컴포넌트 그 자체를 지칭할 때만 사용 | 일반 문맥에서는 "기능 영역" 사용 |
| **우측 레일** | iframe 내부에 임베드되는 `<aside class="bp-right-rail">` | 플랫폼 2패널 중 **"우측 기능 패널"은 폐기됨(2026-04-13)**. 설명은 HTML 자체에 임베드 |

## 버전 체계

| 공식 용어 | 정의 | 비고 |
|-----------|------|------|
| **버전** | Git 태그 1건에 대응 (`v1.0`, `v2.0` 등) | "태그"와 1:1 |
| **최신(미발행)** | 기본 브랜치(main) HEAD를 가리키는 가상 버전 | DB 값 `'latest'`. UI에서는 "최신(미발행)" 문자열 그대로 |
| **메이저** | `v1.x → v2.0` 발행 (구조 변경) | |
| **마이너** | `v1.x → v1.y` 발행 (피드백 반영) | |
| **이월 코멘트** | 미해결 상태로 `version='latest'`에 남아 다음 발행까지 넘어가는 코멘트 | |

## 통합·인증

| 공식 용어 | 정의 | 비고 |
|-----------|------|------|
| **OAuth App** | 내부 사용자 로그인용 Supabase Auth 제공자 | 토큰: 사용자 OAuth 토큰 |
| **GitHub App** | 시스템 행동(파일 읽기·Webhook·태그)용 설치형 앱 | 토큰: Installation 토큰 (1h 자동 재발급) |
| **provider_token** | Supabase OAuth 콜백에서 최초 반환되는 GitHub 토큰 | refresh 시 재반환 안 됨 |
| **초대 링크** | 게스트 초대용 토큰 URL. 7일 만료 | `invitations` 테이블 |

## 권한·상태 표기

| 공식 용어 | 정의 |
|-----------|------|
| **권한 없음** | collaborator가 아닌 사용자가 프로젝트 접근 시 표시되는 화면. [forbidden](./screens/auth/forbidden/screen.md) |
| **승인 대기** | 게스트가 초대 링크로 등록했으나 오너 승인 전인 상태 (`invitation_members.status = 'pending'`) |
| **승인됨** | 오너 수락 후 상태 (`invitation_members.status = 'approved'`) |

**금지 변종**: "403 페이지" → "권한 없음 화면"으로 표기. HTTP 상태 코드와 UI 화면을 혼동하지 않기 위함.
