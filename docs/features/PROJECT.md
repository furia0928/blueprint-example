# PROJECT (프로젝트)

플랫폼의 기본 작업 단위. GitHub 저장소 하나당 하나의 프로젝트가 대응된다.

## INFO (프로젝트 기본 정보)

**rules**
- 프로젝트는 이름, GitHub 저장소(`owner/repo`), 생성자, 생성일, 마지막 발행 정보(태그 SHA, 발행 시각)를 가진다
- 프로젝트 이름은 기본값으로 저장소명(repo)을 사용하며, 이후 사용자가 자유롭게 변경할 수 있다
- 이름은 비어 있을 수 없다
- GitHub 저장소는 한 번 연결되면 변경할 수 없다 (재연결이 필요하면 새 프로젝트 생성)
- 하나의 GitHub 저장소는 하나의 프로젝트에만 연결된다 (중복 연결 불가)
- 마지막 발행 정보는 버전 발행이 있을 때만 설정되며, 없으면 비어 있다

## FAVORITE (즐겨찾기)

**rules**
- 즐겨찾기는 사용자, 프로젝트, 즐겨찾기 시각을 가진다
- 사용자별로 독립적이다 — 한 사용자의 즐겨찾기는 다른 사용자에게 영향을 주지 않는다
- 같은 사용자·프로젝트 쌍은 한 번만 존재한다
- 즐겨찾기된 프로젝트는 목록에서 상단에 정렬된다

## STATS (프로젝트 통계)

**rules**
- 프로젝트 통계는 화면 수, 미해결 코멘트 수를 가진다
- 화면 수는 GitHub 저장소의 디렉토리 구조를 파싱해 계산하며 캐싱한다
- 미해결 코멘트 수는 `open` 상태 코멘트의 개수로 계산된다
- 통계는 프로젝트 카드·목록에서 요약 정보로 사용된다

## ACCESS (접근 권한)

**rules**
- 프로젝트 접근 권한은 GitHub 저장소의 collaborator 관계에 종속된다
- 저장소 collaborator만 프로젝트 목록에서 해당 프로젝트를 볼 수 있다 (public 저장소여도 collaborator가 아니면 미노출)
- collaborator가 아닌 사용자가 프로젝트 URL로 직접 접근하면 권한 없음 화면이 표시된다
- 멤버 정보는 DB에 저장하지 않고 GitHub API로 매번 조회한다 (SSOT는 GitHub)
- Admin 권한은 GitHub 저장소 Admin 권한을 따른다 — 삭제 등 위험 액션에 요구된다
- 오너는 `projects.created_by` 사용자이며, 게스트 초대 기능에 권한이 부여된다 (AUTH 도메인 참조)
- collaborator에서 제거된 사용자가 GitHub에 남긴 이슈/코멘트는 유지된다

## CREATE (프로젝트 생성)

**rules**
- 프로젝트 생성은 선택된 저장소, 프로젝트 이름, 생성자, GitHub App 설치 상태를 가진다
- 프로젝트 생성 전에 대상 저장소에 GitHub App이 설치되어 있어야 한다 (미설치 시 설치 유도)
- 이미 다른 프로젝트에 연결된 저장소는 다시 선택할 수 없다
- 생성자는 저장소 collaborator여야 한다
- 생성 직후 프로젝트 상세 페이지로 자동 이동한다
- 프로젝트 이름의 기본값은 저장소명이며 생성 시 수정할 수 있다

## DELETE (프로젝트 삭제)

**rules**
- 프로젝트 삭제는 GitHub 저장소 Admin 권한을 가진 사용자만 수행할 수 있다
- Admin이 아닌 사용자에게는 삭제 UI 자체가 노출되지 않는다
- 삭제는 확인 입력(프로젝트명 타이핑)을 통과해야 실행된다
- 삭제 시 플랫폼 데이터만 제거된다:
  - `projects` row 삭제
  - `comments` cascade 삭제
  - `github_cache` 해당 프로젝트 캐시 삭제
- GitHub 저장소와 이슈는 유지된다 (플랫폼이 외부 저장소를 건드리지 않는다)
- 삭제 완료 후 프로젝트 목록으로 이동한다

## EXAMPLE (예제 프로젝트)

**rules**
- 예제 프로젝트는 기존 예제 저장소(`blueprint-emotion/blueprint-example`)를 사용자 계정으로 fork하여 생성한다
- 사용자가 프로젝트 0개일 때만 예제 안내가 노출된다
- fork 기반이므로 코멘트(GitHub Issue)가 사용자별로 완전히 격리된다
- fork 완료 후 자동으로 DB 프로젝트 row가 생성되어 목록에 노출된다
- 예제로 생성된 프로젝트 카드에는 "예제" 뱃지가 표시된다
- 한 번 fork하면 프로젝트가 존재하므로 예제 안내는 자연스럽게 사라진다 (별도 거절 플래그 없음)
